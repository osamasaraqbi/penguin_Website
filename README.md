## Overview

**Penguin Studio** is a premium product website for an AI-powered build dashboard designed to help teams move from prompt, concept, and product direction into structured build workflows for **Unity**, **WebGL**, **Android**, and **XR / spatial computing** experiences.

The website presents Penguin Studio as a serious product platform, not a generic landing page. It focuses on the operating flow behind the product: command input, AI agent runtime, Unity handoff, output preview, credit usage, investor readiness, and contact paths for demos, prototype builds, and partnerships.

> **Core message:** Operate every build from one live dashboard.

---

## What the Website Communicates

The site is built to explain Penguin Studio through a clear product-first narrative:

- **AI command dashboard** — a cockpit-style interface for build orchestration.
- **Prompt-to-build workflow** — from user intent into structured build direction.
- **Unity / WebGL / XR output path** — positioning the platform for real interactive production.
- **Agent pipeline** — planner, runtime, tool routing, and build-stage handoff narrative.
- **Credit-based usage model** — a future-ready commercial layer for measurable service usage.
- **Investor and partner readiness** — polished sections for demos, pilots, incubation, and strategic conversations.

---

## Visual Direction

The website uses a premium dark interface style inspired by modern AI dashboards and spatial computing products:

- Deep black / graphite base
- Gold brand accents
- Subtle green runtime indicators
- Glass panels and soft glow layers
- Animated command-system visuals
- Dashboard-style cards, metrics, and cockpit sections
- Mobile-focused responsive cleanup

The visual style is intended to feel **technical, premium, investor-ready, and product-driven**.

## Local Preview

Open `index.html` directly in a browser, or run a simple local server from the repository root:

```bash
python3 -m http.server 8080
```

Then open:

```text
http://localhost:8080
```

## Deployment

The production site is expected to be served through GitHub Pages or the connected deployment flow for:


```bash

git commit -m "update Penguin Studio website"
git push origin main
```
