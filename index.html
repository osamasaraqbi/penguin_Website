Credit System

Credits measure service usage without turning the website into a token page.

Credits are framed as internal service units for AI commands, scene generation, preview builds, API imports, asset review, and enterprise workflows. The protocol layer stays secondary until governance, security, and deployment readiness are complete.

Usage Metering

Credits measure service access inside the dashboard: prompt processing, scene generation, build jobs, previews, template unlocks, and enterprise workflow runs. Each action can show an estimate before submission and record a clean usage event after completion.

price.preview → job.approval → ledger.entry → balance.after

Ledger Discipline

The ledger model is append-only, idempotent, account-scoped, and non-transferable. Failed deterministic jobs can be represented with refunds or reversals, not silent edits to historical records.

grant · purchase · usage · refund · adjustment · migration_reservation
